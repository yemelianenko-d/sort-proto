# Blocks — гайдлайн левел-дизайну (v8)

> **v8 — модель «зони напруги» (виміряно `tools/blocks/diagnose.ts`):**
> 1. **Пазл = щільність дошки.** Порожня дошка нудна (все влазить) і
>    закінчується раптовою смертю. Кожен рівень СТАРТУЄ на авторському
>    симетричному патерні (25–44%, майже-повні лінії, ніколи повна) —
>    перший хід уже пазл, зачистки з першого батчу, дошка живе в зоні
>    ~45–55% заповнення. Бібліотека `PATTERNS` у білдері (cradle, checker2,
>    comb, steps, frame, walls, diamond, gate — драбина щільності).
> 2. **Кожен пул — міні-головоломка**: генерація на користь
>    `SOLVABLE_AFTER_CLEAR` (частка росте з бандом 0.38→0.68) — часто третя
>    фігура влазить лише після зачистки. Це ядро петлі BB.
> 3. **Цілі casual-відкалібровані** (`calibrate.ts` тепер casual-гравець:
>    бере зачистку коли бачить, інакше неідеальне місце; win-rate
>    TUTORIAL 0.9 / EASY 0.8 / NORMAL 0.65 / HARD 0.5 / PEAK 0.4).
>    Щільна дошка складніша → цілі НИЗЬКІ (драбина score 80→150).
>    Рівень = короткий напружений спринт ~8–15 фігур.
> 4. **bigstart лише на відкритих патернах** (cradle) — великі фігури на
>    щільному патерні несправедливі.
> 5. Марафони з квотами 3–4×3–5 — справжні марафони (квоти 8+) чекають
>    progress-persist (фейл зберігає зібране, скидає дошку) — беклог.



> **v7 — головні висновки (з симуляції `tools/blocks/calibrate.ts`):**
> 1. **Керована генерація, НЕ honest.** Чесна генерація (без гарантії
>    розв'язності) на 8×8 занадто жорстока — навіть притомний автоплеєр
>    помирає за ~30-40 ходів. Тому NORMAL+ лишаються КЕРОВАНИМИ (висока
>    частка `SOLVABLE_NOW`), а складність — від розміру цілі + великих фігур
>    (tierMix) + малої частки DANGEROUS. Ранні «добрі» рівні і були
>    керованими — це й є правильна модель.
> 2. **Варіативність = виживання.** Один ШИРОКИЙ master-ростер (усі тетроміно
>    L/J, S/Z, T, I, O + кути + прямокутники, БЕЗ S1) — у випадковому лотку
>    майже завжди щось влазить. ~60% розіграшів тепер кутові/T/S-Z/L (було ~0).
> 3. **Реальна стеля score ≈ 350** на нашому 8×8 при чесній генерації. Драбина
>    130 → ~330, НЕ 1650. Референсні 1960 вимагають більшої дошки або
>    інфляції скорингу — свідомо не женемося. Числа калібруються під
>    win-rate банду автоплеєром, не «на око».
> 4. **Великі фігури на score-рівнях ШКОДЯТЬ** (заповнюють дошку → смерть).
>    Score → master-ростер; bigstart лише для явних «Big Blocks» новинок.

## Принципи (чому референс працює)

1. **Цікавість = частота значущих подій за хвилину** (зачистка, збір, «чи
   влізе?»), а не розмір чисел у цілі. Кожен хід має щось підтверджувати:
   лічильник цокнув, лінія зникла, бар стрибнув.
2. **Порожнеча = безпека, великі фігури = темп.** Ранні дошки ~90–95%
   порожні (лише авторські пресети), а лоток великий (3×3 — норма з L1).
   Поле наповнюється за 5–7 ходів БЕЗ ризику смерті. Щільність приходить
   від гри, не від старту.
3. **Складність ранніх рівнів = розмір квоти. НЕ тісна геометрія.**
   Тісні/щільні старти — інструмент пізніх бандів (HARD+), де програш
   легітимний. Смерть на туторіалі — завжди помилка дизайну.
4. **Перший батч ніколи не буває незручним**: на L1–L6 він авторський
   (`openingBatches`) і тривіально розміщується на порожній дошці.
5. **Спешли течуть через фігури** (до 2 на фігуру, і по 2 однакових теж
   можна) майже щобатчу; пресети на дошці дають ~30–50% квоти. Гравець
   збирає щось майже кожною зачисткою.
6. **Довжина — вісь складності пізньої гри**: марафон = велика квота ×
   виживання під чесним рандомом. Але марафон без густого потоку
   спешлів — гринд; mp2/mb3 обов'язкові.

## Структура кампанії (петля ~40 рівнів)

- **L1–2 — score-онбординг** (референс відкривається двома score).
  Порожня дошка, великі фігури, авторські перші батчі. L1 target = 130
  (виміряна референс-точка).
- **L3–5 — каскад символів**: {0} → {0,1} → {0,1,2}. Дошка порожня +
  4–8 симетричних пресетів (частина квоти). Великі фігури, спешли на
  фігурах з першого батчу.
- **L6 — «Big Blocks» wow**: score, ростер-герой bigstart (3×3 лідер).
- **L7+ — чесна генерація** (NORMAL+): без гарантії розв'язності, рівень
  можна програти. Інтро символів 3, 4 (продовження каскаду). EASY-передишки
  всередині — гарантовані.
- **Score-кластер** (3 поспіль, ~L31–33) — зміна діяльності, як у референсі.
- **Марафони L34+**: 2–5 типів × 8–10, chance 0.8–0.9, mb3/mp2.
- **Петля L41+**: ті самі шаблони з масштабованими числами (референс
  закільцьовується після ~40; каскад символів — маркер початку петлі).

## Дошки

- L1–6: порожні або ЛИШЕ авторські симетричні пресети (4–8 тайлів).
  Заборонено: runway-рядки, random-scatter, будь-що >20% заповнення.
- L7–14: пресети + легкий детермінований scatter ≤8 (кап ≤5 на лінію).
- L15+: scatter 10–14. Пресети завжди симетричні (дзеркало по X або
  точкова симетрія) — авторський почерк референсу.

## Фігури

- Великі фігури — база, дрібнота — приправа. Середній розмір ≥4 клітинок
  у будь-якому ростері. R3x3 — повноцінний громадянин з L1 (tierMix
  TUTORIAL demanding ≥0.13; hero-рівні — BIG_MIX 0.32).
- tierMix множиться на ваги ростера: ростер без DEMANDING-фігур робить
  demanding-частку мертвою — і навпаки. Перевіряти ефективний розподіл.

## Скоринг (виміряно з відео, кадр-в-кадр)

- Постановка = кількість клітинок ✓ (збігається з референсом).
- База зачистки: 1 лінія = 10, трикутна за L (10/30/60) ✓.
- Комбо: множник = довжина ланцюга, кап ×8 — СВІДОМО щедріше за референс
  (там адитивний бонус ~+8–10/крок; наші таргети відкалібровані під наш
  множник). Ланцюг рветься ходом без зачистки, переживає refill.
- Драбина score-таргетів: 130 → 190 → 260 → 380 → 480 → 560 → 680 → 800
  → 950 → 1100 → 1250 → 1400 → 1650.
- Комбо-напис — ТІЛЬКИ ціле число ланцюга («Комбо 3»), без множника.

## Економіка цілей

- Квоти: 1 тип 4–6 → 2 типи 4+4 → 3 типи 5–6 → марафони 8–10×2–5.
- `targetBatchChance` 0.8+ ранні, 0.7 мід, 0.8–0.9 марафони;
  mp2 всюди, mb2 ранні / mb3 пізні. Pity 3, rescue увімкнені.
- Інваріант (перевіряє бейкер): supply (пресети+бюджет) ≥ квота.

## Знахідки референсу, ще не впроваджені

- Progress-persist: фейл марафону зберігає зібране, скидає дошку.
- Фінальний «злив»: залишок квоти ≤3 → знімати кап спешлів на фігуру
  (повністю-спешлові фігури, як H5×5 у відео).
- Веселкова хвиля мульти-зачистки ✓ зроблено; прев'ю лінії «цілими»
  тайлами ✓ зроблено; похвальні написи ✓; прогрес-бар ✓.
