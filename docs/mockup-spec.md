# Технічні вимоги до мокапів: телефон / планшет / десктоп

Доповнення до `asset-spec.md`. Це вимоги до **макетів екранів** (UI/UX),
щоб їх можна було перенести в гру 1:1. Гра рендериться на канвасі, який
завжди дорівнює вікну браузера (без чорних смуг), і лейаут перераховується
під будь-який розмір — тому мокапи мають показувати не один «ідеальний»
екран, а поведінку інтерфейсу в діапазоні пропорцій.

## 1. Базові канви мокапів

Усі розміри — **логічні пікселі** (як у Figma-фреймах). Графіка всередині
експортується @2x, але самі макети робимо в 1x логічних розмірах.

| Форм-фактор | Базовий фрейм | Обов'язкові перевірочні фрейми | Орієнтація |
|---|---|---|---|
| Телефон | **390×844** | 360×640 (малий Android), 430×932 (великий iPhone) | Портрет — основна. Ландшафт 844×390 — окремий макет (див. §5) |
| Планшет | **834×1194** (портрет) | 1194×834 (ландшафт), 768×1024 | Обидві |
| Десктоп | **1440×900** | 1920×1080, 1280×720 | Ландшафт |

Мінімальна логічна ширина, яку гра підтримує: **360 px**. Усе, що працює
на 360×640, працюватиме скрізь.

## 2. Сітка лейауту гри (однакова для всіх платформ)

Ігровий екран ділиться на три горизонтальні зони; мокап має їх поважати:

```
┌─────────────────────────────┐
│ HUD (висота 64)             │ ← рівень/ціль, ходи, назад, рестарт
├─────────────────────────────┤
│ Ігрове поле (flex)          │ ← колонки; центруються, 1–3 ряди
├─────────────────────────────┤
│ Панель бустерів (висота 84) │ ← назад-хід, лупа, ключ
└─────────────────────────────┘
```

- Поле саме розкладає колонки: розмір кубика 30–64 px, від 1 до 3 рядів
  колонок залежно від ширини. У мокапах показати два кейси:
  **5 колонок** (рівні 1–3, один ряд) і **9 колонок** (рівень 10,
  2 ряди на телефоні / 1–2 на планшеті й десктопі).
- Бічні панелі (як «Підказка по механіці» в затвердженому мокапі) —
  **тільки** планшет-ландшафт і десктоп, ширина панелі 280–320 px,
  поле лишається в центрі. На телефоні їх немає.

## 3. Safe areas (телефон — обов'язково)

Портретний макет телефона повинен мати позначені зони, куди не можна
класти інтерактив:

| Зона | Відступ |
|---|---|
| Верх (челка/статус-бар) | 59 px |
| Низ (home indicator) | 34 px |
| Боки в ландшафті | по 47 px |

HUD і панель бустерів у грі автоматично зсуваються на ці відступи —
у мокапі просто показати фрейм з увімкненими safe-area guides
(в Figma — «iPhone 14» layout guides або прямокутники-плейсхолдери).

## 4. Інтерактив і типографіка

- **Мінімальний тап-таргет: 44×44 px** (включно з невидимим паддінгом).
  Кнопки бустерів у грі зараз 76×52, іконки-кнопки 44×44 — менше не робити.
- Мінімальний кегль тексту: 14 px на телефоні, 16 px на десктопі.
  Основні шрифти з кирилицею фіксуємо в макеті (зараз у грі: Caveat —
  заголовки/цифри, Neucha — текст; якщо пропонуєте інші рукописні з
  повною кирилицею — вкажіть назви і ліцензію).
- Увесь текст у макетах — живий текст (не растр), бо гра рендерить його
  шрифтом; з макета беремо стилі, а не картинки.
- Контраст тексту до фону — не нижче WCAG AA (4.5:1) для основного тексту.

## 5. Що саме мокапити (мінімальний набір)

| Екран | Телефон портрет | Телефон ландшафт | Планшет (порт+ланд) | Десктоп |
|---|---|---|---|---|
| Лоббі (лого, карта рівнів, прогрес) | ✅ | спрощено* | ✅ | ✅ |
| Геймплей: 5 колонок | ✅ | ✅ | ✅ | ✅ |
| Геймплей: 9 колонок + прихованi блоки + замкнена колонка | ✅ | — | ✅ | ✅ |
| Стани колонки: звичайна / вибрана / валідна ціль / замкнена | ✅ (один аркуш станів, спільний для всіх) | | | |
| Попапи: перемога (1–3★), «немає ходів», туторіал | ✅ (спільні, масштабуються) | | | |
| Екрани завантаження і помилки | ✅ (спільні) | | | |

\* Ландшафт телефона: достатньо показати принцип (HUD лишається зверху,
бустери можуть переїхати праворуч колонкою або лишитись знизу — рішення
за дизайном, але зафіксувати).

## 6. Правила адаптації (зафіксувати в макетах анотаціями)

Для кожного плаваючого елемента макет має відповідати на питання
«до чого прив'язаний і як тягнеться»:

- HUD: назад — до лівого краю + safe, рестарт — до правого + safe,
  заголовок — центр/зліва.
- Поле: завжди центроване по обох осях у своїй зоні.
- Попапи: фіксована ширина min(320, екран−40), центр екрана.
- Бічна панель (планшет-ланд/десктоп): прив'язана до правого краю,
  при звуженні вікна зникає першою (behavior: hide, не squeeze).
- Фон: безшовний тайл на весь екран; декор «на полях» (дірки біндера,
  червона лінія, дудли) — окремий шар, прив'язаний до країв екрана,
  на телефоні — зменшений набір, щоб не конфліктувати з полем.

## 7. Формат поставки мокапів

- **Figma** (бажано): компоненти зі станами (кнопки: default/pressed/
  disabled; колонки: 4 стани), auto-layout там, де списки; фрейми названі
  за схемою `phone/gameplay-5cols`, `tablet-land/lobby`, `desktop/win-popup`.
- Або пошарові PSD з тими самими іменами фреймів.
- До макетів — сторінка «Redlines»: відступи зон, розміри кнопок, стилі
  тексту (шрифт/кегль/колір у hex).
- Прев'ю-експорт PNG кожного фрейму в 1x — щоб швидко дивитись без Figma.

## 8. Чекліст приймання мокапів

- [ ] Усі базові фрейми з §1 присутні
- [ ] На 360×640 нічого не обрізано і тап-таргети ≥44 px
- [ ] Safe areas позначені й дотримані на телефоні
- [ ] Показані обидва кейси поля (5 і 9 колонок)
- [ ] Є всі 4 стани колонки і 3 попапи
- [ ] Текст живий, шрифти з кирилицею вказані
- [ ] Бічні панелі відсутні в телефонних фреймах
- [ ] Є Redlines-сторінка з розмірами і стилями
